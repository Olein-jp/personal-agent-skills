# Codex Issue Templates

Use these templates when the user asks for a ready-to-use GitHub Issue body. Keep or remove sections based on task size.

## S: Small Fix

````markdown
# 目的

`path/to/file` のみを主な対象として、[現在の問題] を [期待する状態] に変更してください。

## 作業対象

最初に以下だけを確認してください。

- `path/to/file`
- `tests/path/to/related-test` <!-- ある場合のみ -->

上記から直接参照されているコードは、実装に必要な場合のみ追加で確認してください。リポジトリ全体の調査は不要です。

## 受け入れ条件

- [ ] [期待する挙動]
- [ ] [既存の関連挙動] が壊れていない
- [ ] 関連するテストまたは確認が成功する

## 制約・対象外

- 関係のないリファクタリングは行わない
- ファイル構成や公開APIは変更しない
- 依存パッケージは更新しない

## 検証

以下の関連確認だけを実行してください。

```bash
# 例: composer test -- tests/Unit/TargetTest.php
```

## 作業方法

- サブエージェントは使用しない
- 詳細な計画書は作成せず、関連コードの確認後に実装する
- 受け入れ条件を満たした時点で作業を終了する

## 完了報告

以下だけを簡潔に報告してください。

1. 変更内容
2. 変更ファイル
3. 実行した検証と結果
4. 未解決事項
````

- [確認したい既存処理]
- [呼び出し関係または保存先]
- [既存テストの書き方]

## 実装要件

- 
- 
- 

## 受け入れ条件

- [ ] 
- [ ] 
- [ ] 
- [ ] 関連する既存テストと追加テストが成功する

## 制約

- 既存の公開APIを変更しない
- 既存の設計、命名規則、コーディング規約に従う
- 変更範囲を必要最小限にする

## 作業方法

- このタスクではサブエージェントを使用しない
- 詳細な計画書は作成せず、関連コードの確認後に実装する

- 追加の改善案は実装せず、必要なら未解決事項として報告する

## 完了報告

1. 変更内容
2. 変更ファイル
3. 実行した検証と結果
4. 未解決事項

## L: Investigation First

````markdown
# 目的

[大きめの変更で実現したい結果] について、今回は実装せず、影響範囲と実装計画を整理してください。

## 調査対象

最初に以下を確認してください。

- `path/to/entry`
- `path/to/related`
- `docs/path/to/spec` <!-- 必要な場合のみ -->

上記から直接参照されているコードは、調査に必要な場合のみ追加で確認してください。リポジトリ全体の網羅的調査は不要です。

## 調査で確認したいこと

- 現在の処理の入口と責務
- 変更が必要になりそうなファイル
- 既存テストと不足しているテスト
- 後方互換性、データ移行、セキュリティ、UI への影響

## 対象外

- 今回は実装しない
- コード変更、依存更新、フォーマット修正は行わない
- 網羅的なリポジトリ調査は行わない

## 成果物

以下を簡潔に報告してください。

1. 現状理解
2. 推奨する実装方針
3. 変更予定ファイル
4. 必要なテスト・検証
5. リスク・確認事項

実装計画は 5 項目以内にしてください。
````

## WordPress Additions

Add only when relevant:

```markdown
## WordPress 固有の要件

- WordPress Coding Standards に従う
- 入力値をサニタイズする
- 出力時に文脈に応じてエスケープする
- 管理画面操作では権限チェックと nonce 検証を行う
- 翻訳対象文字列には適切な i18n 関数を使用する
- 既存フックと公開APIの互換性を維持する
- 独自実装より WordPress 標準APIを優先する
- アクセシビリティを損なわない
```
